代码修改内容请严格遵守以下规则
1. 这些修改内容将由一个智能较低的模型快速应用。你需要明确指出具体修改的内容，同时尽量减少不变代码的书写。  
在编写修改内容时，应该依次列出每个修改，并使用特殊注释 `// ... existing code ...` 来表示编辑行之间未更改的代码。

// ... existing code ...

// ... existing code ...
2.修改的代码行应包含足够的未变代码上下文，以消除歧义。
3. 绝对不要省略原有代码（或注释）片段而不使用 // ... existing code ... 注释来标明其缺失。如果你省略了该注释，模型可能会误删这些行。
