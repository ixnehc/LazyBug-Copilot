# LazyBug - Visual Studio AI 编程助手插件

## 产品概述

**LazyBug** 是一款专为 Visual Studio 设计的"cursor-like"智能编程助手插件，融合大语言模型（LLM）能力，为开发者提供智能化的代码创建、重构和问答体验。插件支持多种主流 AI 服务提供商，让开发者在熟悉的 IDE 环境中享受 AI 辅助编程的便利。

---

## 核心功能

### 1. 智能对话系统

- 支持基于 Markdown 的多轮对话展示, 支持 代码高亮、Diff 视图等富媒体展示,并可自由缩放字体大小
- 支持创建新会话、切换历史会话
- 可一键回滚到会话的任意历史状态,并可取消回滚
- 会话成本统计展示
- 上下文使用率可视化进度条
- 对话历史按项目存储,可持久化保存

### 2. 代码智能编辑

- AI 可以直接修改项目中的文件内容,可一次修改多个文件
- 可在代码编辑窗口查看修改前后对比视图（Diff View）
- 文件修改追踪与回滚
- 文件备份系统,在万一回滚失效(比如系统bug)时,仍可找回用户原始文件

### 3. 代码数据库搜索

- 基于VisualStudio项目文件的高效搜索
- 支持超大型项目的快速文本搜索
- 支持超大型项目的快速符号查询(目前仅支持c/c++)
- AI 可自动读取相关文件内容获取上下文

### 4. 智能输入框

- 使用标签系统管理附加文件
- 自动补全：@符号触发文件/符号的自动补全
- PageUp/PageDown 快速浏览输入历史
- 快速切换大语言模型

### 5. 多模型支持

- 默认API 提供商
	- OpenAI (GPT-4, GPT-3.5 等)
	- Anthropic (Claude 系列)
	- Google (Gemini 系列)
	- OpenRouter (多模型聚合)
	- Moonshot AI(Kimi)
	- z.ai (GLM)
- 可修改配置文件支持自定义的API 端点
- 支持OpenAI兼容,Anthropic,Gemini 三种api格式

## 使用场景

- **代码审查与重构**：让 AI 分析代码并提出重构建议
- **新功能开发**：描述需求，AI 协助生成代码框架和实现
- **Bug 修复**：描述问题现象，AI 帮助定位并修复 Bug
- **代码解释**：询问复杂代码逻辑，AI 提供详细解释

## 使用Tips

- **打开对话面板**：通过 Visual Studio 菜单 `View` -> `Other Windows` -> `LazyBug Chat` 可打开对话面板。建议为该命令（`View.LazyBugChat`）绑定快捷键，以便随时呼出。
- **数据库与磁盘空间**：LazyBug Chat 的数据库独立于项目，统一存放在 C 盘。请确保 C 盘有充足的可用空间（对于超大型项目，可能需要预留 10GB 以上空间）。
- **查看备份与数据库目录**：将鼠标悬停在对话面板右上角的“设置”按钮上片刻，会显示目录图标。点击即可浏览当前项目的数据库目录。AI 最近修改过的文件会自动在该目录下的 `_backup` 文件夹中进行备份。
- **添加文件附件**：
  - 在 `Solution Explorer` 面板中选中文件，右键点击并选择 `Add to LazyBug Chat`，即可将其作为附件加入当前对话。
  - 在代码编辑器上方当前文件的标签页（Tab）上右键，也可以执行相同操作。
- **自动搜索与上下文**：对话窗口内的附件会被完整提交给大模型。但在多数情况下，您无需手动添加代码文件为附件。只需在对话中提供相关关键字，AI 就会自动搜索并读取所需的文件内容。
- **符号数据库构建（C/C++）**：首次打开 C/C++ 项目时，LazyBug 会在后台自动构建符号数据库。对于超大型项目（例如 300 万行代码），此过程可能需要较长时间（约 30 分钟至 1 小时）。在构建完成前，符号查询结果可能会不够准确。
- **代码对比（Diff 视图）**：
  - 点击对话面板中文件编辑块的标题，可在主编辑区展示修改前后的 Diff 视图；按下空格键可取消显示。
  - 连续点击文件编辑块的标题，可以快速在不同的 Diff 修改段落间进行跳转浏览。
- **避免编辑冲突**：在 AI 正在工作（尤其是正在修改文件内容）时，请勿手动编辑文件。
- **界面缩放**：当鼠标焦点位于 LazyBug 对话窗口内时，按住 `Ctrl` 键并滚动鼠标滚轮，可自由缩放界面及文字大小。
- **自定义 API 与 Prompt Cache**：在自定义 LLM API 配置时，若供应商支持，建议优先使用原生 API 格式（特别是 Anthropic 模型），以确保 Prompt Cache（提示词缓存）功能正常生效。
- **消费统计说明**：每轮对话的消费金额是根据您在配置表中填写的单价计算得出的。当大模型提供商采用复杂的计费模式（如部分订阅计划）时，该统计结果仅供参考。
- **上下文进度条与限制**：
  - **进度条含义**：反映最近一轮对话的上下文占用情况。当进度条变红时，表示上下文即将耗尽，大模型的输出质量可能会下降（如出现“幻觉”现象）。
  - **上限设置**：进度条的上限可在 LLM 的 API 配置文件中设定，代表模型能保持高质量输出的最大上下文长度（通常应设置为略小于模型标称的最大容量）。
  - **使用建议**：当进度条变红时，请尽量避免在当前会话中继续对话（目前插件暂无上下文压缩功能），建议开启新会话。
- **任务拆分策略**：LazyBug 并非为处理超大型复杂任务而设计。请尽量将您的开发任务拆解为粒度较小、目标明确的子任务，以获得最佳的 AI 辅助体验。
