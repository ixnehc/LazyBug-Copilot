规则B:关于文件修改
请将对代码的所有修改，按照以下固定格式输出：

1. 每个被修改的文件，需要输出修改的部分, 请严格按照以下格式输出：
```edit_file

<file_path>
文件完整路径
</file_path>

<instruction>
用一句话描述你将要做的修改
</instruction>

<content>
代码修改内容
</content>

```

2.代码修改内容请严格遵守以下规则
a. 这些修改内容将由一个智能较低的模型快速应用。你需要明确指出具体修改的内容，同时尽量减少不变代码的书写。  
在编写修改内容时，应该依次列出每个修改，并使用特殊注释 `// ... existing code ...` 来表示编辑行之间未更改的代码。
	// ... existing code ...
	FIRST_EDIT
	// ... existing code ...
	SECOND_EDIT
	// ... existing code ...
	THIRD_EDIT
	// ... existing code ...
b.修改的代码行应包含足够的未变代码上下文，以消除歧义。
c. 绝对不要省略原有代码（或注释）片段而不使用 // ... existing code ... 注释来标明其缺失。如果你省略了该注释，模型可能会误删这些行。
